## ✅ Permitted Uses

### Open Source License Compliance

While the source code is licensed under MIT License, the trademarks remain protected. You may:

1. **Use the software** as licensed under MIT
2. **Distribute unmodified copies** with original attribution
3. **Reference the project name** in educational or informational contexts
4. **Link to the official repository** using the official name

### Acceptable References

- "Based on OYI - Open in your IDE by Orvexa by KAGEYOSHI"
- "Compatible with OYI - Open in your IDE"
- "Inspired by the OYI project from Orvexa by KAGEYOSHI"
- "This extension extends the functionality of OYI"

## ❌ Prohibited Uses

### Trademark Violations

You may **NOT**:

1. **Use any Orvexa trademarks** in your product or service names
2. **Imply endorsement** or official affiliation without permission
3. **Use confusingly similar names** that could cause consumer confusion
4. **Register domain names** containing these trademarks
5. **Create derivatives** that use the protected trademarks
6. **Modify the trademarks** or create variations

### Examples of Prohibited Use

- ❌ "Orvexa IDE Manager"
- ❌ "KAGEYOSHI Code Tools"  
- ❌ "OYI Plus" or "OYI Pro"
- ❌ "Unofficial OYI Extension"
- ❌ Using Orvexa logos in your applications

## 🔄 Forking and Distribution Guidelines

### For Forks and Derivatives

When creating forks or derivative works:

1. **Remove all trademark references** from:
   - Product names and descriptions
   - User interface elements
   - Documentation and marketing materials
   - Package names and identifiers

2. **Replace with your own branding**:
   - Choose a unique, non-confusing name
   - Create your own visual identity
   - Update all references in code and documentation

3. **Maintain attribution**:
   - Keep copyright notices
   - Include "Based on OYI by Orvexa by KAGEYOSHI"
   - Link to the original project

### Code Example for Forks

```json
{
  "name": "your-unique-ide-opener",
  "displayName": "Your IDE Opener",
  "description": "Your description (based on OYI by Orvexa by KAGEYOSHI)",
  "publisher": "YourName",
  "author": "Your Name",
  "attribution": "Based on OYI - Open in your IDE by Orvexa by KAGEYOSHI"
}
```

## 🛡️ Enforcement

### Monitoring

We actively monitor for trademark violations:

- **Domain registrations** containing our trademarks
- **Software packages** using confusing names
- **Marketing materials** implying false affiliation
- **Social media accounts** misusing our brands

### Enforcement Actions

Violations may result in:

1. **Cease and desist** notices
2. **DMCA takedown** requests
3. **Legal action** for damages
4. **Domain disputes** through UDRP proceedings

### Safe Harbor

We will not pursue enforcement against:

- **Good faith** educational uses
- **Fair use** in reviews and criticism
- **Proper attribution** in academic contexts
- **Compliance** with these guidelines

## 📋 Reporting Violations

### How to Report

If you discover trademark misuse:

1. **Document the violation**:
   - Screenshots or evidence
   - URLs or contact information
   - Date and time discovered

2. **Email us**: orvexa.support@kageyoshi.com

3. **Include**:
   - Description of the violation
   - Evidence of confusion or harm
   - Your contact information

### Response Process

1. **Investigation**: We will review the report
2. **Contact**: We may reach out for additional information
3. **Action**: Appropriate enforcement action will be taken
4. **Follow-up**: You will be notified of the resolution
